Integration Overview

Microsoft Defender for Cloud Apps integrates with Zscaler to provide comprehensive SaaS Security Posture Management (SSPM) by combining Microsoft's cloud app governance with Zscaler's secure web gateway capabilities.

üìã Prerequisites

Microsoft Requirements

  • Microsoft Defender for Cloud Apps license
  • Global Administrator or Security Administrator permissions
  • Active Microsoft 365 or Azure AD tenant

Zscaler Requirements

  • Zscaler Internet Access (ZIA) subscription
  • Zscaler administrator access
  • NSS (Nanolog Streaming Service) enabled

1

Configure Zscaler NSS

Set up Nanolog Streaming Service in Zscaler to send web logs to Microsoft Defender for Cloud Apps

  • Navigate to Zscaler Admin Portal ‚Üí Administration ‚Üí Nanolog Streaming Service
  • Add new NSS feed with Microsoft Cloud App Security endpoint
  • Configure log format to include required fields for SSPM analysis
2

Enable Log Collection in Defender

Configure Microsoft Defender for Cloud Apps to receive and process Zscaler logs

  • Go to Microsoft Defender Portal ‚Üí Cloud Apps ‚Üí Settings ‚Üí Log collectors
  • Create new automatic log upload for Zscaler
  • Verify log parsing and application discovery is working
3

Configure App Discovery

Set up continuous app discovery to identify all SaaS applications in use

  • Enable continuous reports in Cloud Discovery dashboard
  • Configure user group mapping for organizational units
  • Set up automated app risk scoring based on usage patterns
4

Apply SSPM Policies

Create and enforce SaaS Security Posture Management policies across discovered applications

  • Define app categories (Sanctioned, Unsanctioned, Monitored)
  • Create governance policies for file sharing and access controls
  • Set up alerts for policy violations and risky app usage

üîß Troubleshooting

Log Ingestion Issues

Solution: Verify NSS configuration, check firewall rules, and ensure proper authentication

App Discovery Not Working

Solution: Review log format configuration and ensure all required fields are included

Problem: SSPM policies not being applied in real-time

Solution: Check policy configuration and consider adjusting sync intervals
