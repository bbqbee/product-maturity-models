2. üîç Token Theft Attack Anatomy

Understanding the token theft attack chain is crucial for implementing effective defenses. Modern attacks follow a predictable pattern that Microsoft's security solutions are specifically designed to disrupt.

The Attack Chain

Phase 1: Initial Compromise (T+0 to T+5 minutes)

  • Phishing Email: User receives convincing email with malicious link
  • Credential Harvesting: Fake login page captures username/password
  • MFA Bypass: Attacker-in-the-Middle (AiTM) captures MFA token

Phase 2: Token Extraction (T+5 to T+15 minutes)

  • Session Cookie Theft: Browser session tokens extracted
  • Primary Refresh Token (PRT) Theft: Device-level tokens compromised
  • Cloud Authentication Token Theft: Application-specific tokens stolen

Phase 3: Lateral Movement (T+15 minutes to T+24 hours)

  • Token Replay: Stolen tokens used from attacker's infrastructure
  • Resource Access: Email, SharePoint, and cloud applications accessed
  • Data Exfiltration: Sensitive information downloaded or shared

üéØ Why Traditional Defenses Fail

Token-based attacks bypass traditional security measures because:

  • Stolen tokens appear as legitimate authentication
  • No need to crack passwords or bypass MFA
  • Attacks can be executed from any global location
  • Detection requires behavioral analysis, not signature matching

Critical Attack Indicators

  • Anomalous sign-in locations or times
  • Impossible travel scenarios
  • Unusual application access patterns
  • High-volume data download activities
  • Token usage from unregistered devices
