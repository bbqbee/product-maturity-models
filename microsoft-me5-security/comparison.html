‚úÖ GOOD: Entra ID P1 Foundation

Entra ID P1 provides the foundational layer for token protection through basic conditional access policies and device management capabilities.

üîê

Token Protection for Sign-in Sessions

Require token protection in Conditional Access policies for supported applications. Prevents stolen tokens from being replayed on unauthorized devices.

How it works: Cryptographically binds tokens to registered devices using Primary Refresh Tokens (PRTs)
üì±

Basic Device Management

Conditional Access policies requiring managed devices and basic compliance checks.

Coverage: Device registration, basic compliance policies, device-based access controls
üåç

Location-Based Controls

IP-based named locations and basic geographic restrictions for access control.

Benefit: Prevents token replay from unauthorized geographic locations
üì≤

App Protection Policies

Require approved client apps or app protection policies for mobile devices.

Protection: Prevents data access from unmanaged mobile applications

üõ°Ô∏è How App Protection Policies Defend Against Token Theft

App protection policies in Entra ID P1 provide a crucial layer of defense by:

  • Isolating Corporate Data: Creates secure containers within apps, preventing token extraction through malicious apps
  • Blocking Screen Capture: Prevents token harvesting through screenshot malware
  • Requiring PIN/Biometric: Adds additional authentication layer even if device is compromised
  • Detecting Jailbroken Devices: Blocks access from compromised devices more likely to expose tokens
  • Enforcing App Versions: Ensures only patched, secure app versions can access corporate data

üîß Intune Integration for P1

Device Compliance Policies

Intune enforces device health requirements (OS version, encryption, antivirus) that Conditional Access policies can evaluate before granting token access.

App Protection Policy Management

Centralized creation and deployment of app protection policies that work with Conditional Access to require secure apps for token-protected resources.

Device Configuration

Automatic deployment of security settings (Windows Defender, encryption) that harden devices against token theft malware.
